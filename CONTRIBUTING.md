# Guia de Contribuição

Como contribuir para este repositório de documentação.

## 📝 Adicionando Instruções do Copilot

1. Crie arquivo em `.github/instructions/`
2. Use sufixo `.instructions.md`
3. Seja específico e objetivo
4. Teste com o Copilot antes de commitar

**Exemplo:**
```markdown
---
description: Breve descrição
---

# Título da Instrução

## Seção 1
- Item 1
- Item 2

## Seção 2
- Item 1
- Item 2
```

## 💬 Adicionando Prompts

1. Crie arquivo em `.github/prompts/`
2. Use nome descritivo
3. Inclua contexto e objetivo
4. Teste antes de adicionar

## 📚 Adicionando Documentação

1. Coloque em `docs/` com estrutura apropriada
2. Use markdown
3. Seja claro e prático
4. Inclua exemplos quando útil

## 📋 Atualizando Changelogs

1. Adicione no projeto específico em `changelogs/`
2. Siga formato [Keep a Changelog](https://keepachangelog.com/pt-BR/)
3. Categorize corretamente (Added, Changed, Fixed, etc)
4. Use linguagem clara

## ✅ Checklist

Antes de commitar:

- [ ] Arquivo no diretório correto
- [ ] Usa formato markdown
- [ ] Conteúdo claro e objetivo
- [ ] Testado (se aplicável)
- [ ] README atualizado (se necessário)

## 🔄 Processo

1. Faça suas alterações
2. Commit com mensagem descritiva
3. Push para o repositório

## 💡 Boas Práticas

- Mantenha arquivos pequenos e focados
- Use português claro
- Evite redundância
- Revise periodicamente para atualizar ou remover obsoletos
